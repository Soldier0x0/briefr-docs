# Unit 16: Tradeoff-driven design practice

<CoverageBadge coverage="gap" />

Tradeoff reasoning is not a feature you implement — it is the meta-skill that shapes every feature you do implement. BRIEFR's own design history provides several concrete worked examples: choices made for explicit reasons, with explicit alternatives rejected. This unit maps the topic and points to external resources for deeper practice.

        <li><strong>Monolith over microservices</strong> — deployment simplicity and one-box ops cost vs independent scaling and polyglot services. Rejected because the team and load do not justify the coordination overhead.</li>
