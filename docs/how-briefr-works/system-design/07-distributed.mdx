# Unit 7: Distributed systems basics

<CoverageBadge coverage="gap" />

BRIEFR is a deliberate single-box deployment. The distributed-systems problems that occupy a large portion of system-design literature — consensus, split-brain, clock skew, exactly-once delivery — do not arise when there is only one node. This unit maps the topic for completeness and explains the trade-off BRIEFR makes by staying on one machine.

SQLite's write-ahead log or Postgres's connection pool serializes concurrent access at the engine level. There is no cluster to elect a leader from, no network partition between components, and no vector clock needed to order events. The deliberate alternative to distributed coordination is keeping everything on one machine within one process boundary.
