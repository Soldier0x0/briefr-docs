## Gap unit outline

**Topic:** Distributed systems basics — consensus, leader election, clock skew, exactly-once delivery.

**What it covers:**
- Consensus algorithms (Paxos, Raft) and why they exist
- Leader election and split-brain avoidance
- Logical clocks, vector clocks, and clock skew
- Exactly-once delivery guarantees

**Why it matters:** Once a system spans more than one node, many guarantees that feel natural on a single machine (shared memory, synchronized clocks, atomic writes) disappear. Distributed algorithms restore them at the cost of complexity.

**In BRIEFR:** BRIEFR runs on a single box with a single writer. There is no distributed state, no consensus requirement, and no leader election. Local locks and a single scheduler owner are the deliberate alternative.

**Why BRIEFR skips this:** Single self-hosted deployment; failure domain is one box by product choice; ops and cognitive cost of consensus clusters are not justified for a CVE portal.

**When you would need it:** Multiple stateful nodes requiring coordination; active-active multi-region; split-brain scenarios for services that must stay up across node failures.
