# System Design — Learning Pathway

This pathway follows the standard system-design curriculum order — foundations
through design review — and uses BRIEFR as the primary teaching case. Every unit
is tagged so you know exactly what you are getting.

## Coverage legend

| Tag | Meaning |
|---|---|
| <CoverageBadge coverage="briefr" /> | BRIEFR implements this topic. Taught directly from the real product. |
| <CoverageBadge coverage="partial" /> | BRIEFR implements part of this topic. The unit covers BRIEFR first, then points to curated externals for the rest. |
| <CoverageBadge coverage="gap" /> | BRIEFR omits or intentionally limits this topic. The unit explains what it covers, why BRIEFR skips it, and where to learn it externally. |

## How to use this pathway

1. Start with **[Unit 0 — How to use this pathway](/docs/how-briefr-works/system-design/00-how-to-use)**.
2. Work through units **in order** — each one assumes the previous.
3. For **briefr** units: read the BRIEFR chapters linked, then return.
4. For **partial** units: read the BRIEFR section, then decide whether to follow the external links before continuing.
5. For **gap** units: read the rationale and external pointers, then use "Return here" to continue the spine.
6. Cross-read Track B ([How it's built](/docs/how-briefr-works/how-its-built/)) for deeper implementation detail on the `briefr` units.
