# How BRIEFR Works — and Why

This section teaches **how a self-hosted intel data plane uses security data** —
collect, normalize, enrich, correlate, hunt, detect, prioritize — and **how
BRIEFR is engineered** to do that on one box. It is not a general cybersecurity
syllabus. You get fluent in *this* tool, with the real repo as a parallel
textbook.

## Three tracks

| Track | What you learn | Start here |
|---|---|---|
| **Intel lifecycle** | The data's journey from external feeds to the analyst brief | [Collect](/docs/how-briefr-works/intel-lifecycle/collect) |
| **How it's built** | System design as BRIEFR implements it, with notes on how larger orgs differ | [Ingestion and scheduler](/docs/how-briefr-works/how-its-built/ingestion-scheduler) |
| **System Design** | Classic SD curriculum grounded in BRIEFR — honest gap units where the product omits a topic | [System Design pathway](/docs/how-briefr-works/system-design/) |

Not sure which track fits you? Start at **[Pathways](/docs/pathways)** to pick a route
based on your role.

## Rules of the road

- Every chapter answers the same questions: what / why / where in BRIEFR / how / what it needs / how industry differs.
- Deploying BRIEFR is **recommended**, never required.
- The docs site never asks for your API keys and never fires live third-party calls.
