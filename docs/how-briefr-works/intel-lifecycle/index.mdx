# Intel lifecycle

Track A follows security intel from external feeds into your morning brief. Each chapter uses the same template: what it is, why BRIEFR does it, where it lives in the repo, how it works, what it needs, and how industry differs.

## Spine

| Step | Chapter | What you'll learn |
| --- | --- | --- |
| 1 | [Collect](intel-lifecycle/collect) | Scheduler-driven ingest from NVD, KEV, EPSS, and other open feeds |
| 2 | [Normalize](intel-lifecycle/normalize) | Shaping raw payloads into consistent records before enrichment |
| 3 | [Enrich](intel-lifecycle/enrich) | Adding threat context — OTX, exploit refs, ATT&CK, GreyNoise |
| 4 | [Correlate](intel-lifecycle/correlate) | Explainable links between CVEs via the nightly correlation engine |
| 5 | [Hunt](intel-lifecycle/hunt) | Analyst workflows for searching and pivoting on intel |
| 6 | [Detect](intel-lifecycle/detect) | Detection engineering — Sigma rules and Forge output |
| 7 | [Prioritize / report](intel-lifecycle/prioritize) | Scoring, morning brief ranking, and PDF export |
| Reference | [Sources](intel-lifecycle/sources) | Deep-dives for NVD, CISA KEV, EPSS, and AlienVault OTX |

Start at [Collect](intel-lifecycle/collect), inspect the [Sources](intel-lifecycle/sources) behind the first lifecycle stages, or pick a [pathway](pathways) if you already know your role.
