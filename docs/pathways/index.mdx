## Security analyst path

You care about the feed, enrichment, and why a CVE rose in the brief.

1. [Collect](/docs/how-briefr-works/intel-lifecycle/collect)
2. [Normalize](/docs/how-briefr-works/intel-lifecycle/normalize)
3. [Enrich](/docs/how-briefr-works/intel-lifecycle/enrich) — optional [Sources](/docs/how-briefr-works/intel-lifecycle/sources) deep-dives for NVD, KEV, EPSS, and OTX
4. [Correlate](/docs/how-briefr-works/intel-lifecycle/correlate)
5. [Hunt](/docs/how-briefr-works/intel-lifecycle/hunt)
6. [Detect](/docs/how-briefr-works/intel-lifecycle/detect)
7. [Prioritize / report](/docs/how-briefr-works/intel-lifecycle/prioritize)

Then skim [Resilience](/docs/how-briefr-works/how-its-built/resilience) so quota
and rate limits make sense when a source goes quiet.

## Security architect / builder path

You care about how the machine is wired.

1. [Ingestion and scheduler](/docs/how-briefr-works/how-its-built/ingestion-scheduler)
2. [Resilience](/docs/how-briefr-works/how-its-built/resilience)
3. [Storage](/docs/how-briefr-works/how-its-built/storage)
4. [API and auth](/docs/how-briefr-works/how-its-built/api-auth)
5. [Background jobs](/docs/how-briefr-works/how-its-built/background-jobs)
6. [Search and embeddings](/docs/how-briefr-works/how-its-built/search-embeddings)
7. [Webhooks and operations](/docs/how-briefr-works/how-its-built/webhooks-ops)

Cross-read the [Intel lifecycle](/docs/how-briefr-works/intel-lifecycle/) spine so
design choices stay tied to the data path.

## System design path

A classic system-design curriculum — foundations through design review — taught
from BRIEFR where the product maps, with honest gap units where it does not.
Start with [How to use this pathway](/docs/how-briefr-works/system-design/00-how-to-use),
then follow the numbered units in order.

Units are tagged **briefr** (BRIEFR-primary), **partial** (BRIEFR section first,
then externals), or **gap** (BRIEFR omits this by design — rationale + curated
resources).

## Deploy while you learn

Installing BRIEFR on your own hardware is **recommended but never required** for
reading these chapters. When you are ready, start with
[Self-host BRIEFR](/docs/admin-guide/self-host). Production operators use
`briefr-install.sh` / `briefr-deploy.sh` / `briefr-service.sh` — not a daily
git-pull workflow.
